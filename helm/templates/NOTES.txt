
Kure Monitor has been deployed to namespace {{ .Release.Namespace }}.

1. Access the dashboard:

   kubectl port-forward svc/{{ include "kure.fullname" . }}-frontend {{ .Values.frontend.service.port }}:{{ .Values.frontend.service.port }} -n {{ .Release.Namespace }}

   Then open http://localhost:{{ .Values.frontend.service.port }}

2. On first visit, the dashboard will prompt you to create the initial admin
   account (username + password). After signing in you can invite additional
   users (with read or write roles) from the Admin panel.

   Service-to-service authentication (agent and security scanner -> backend)
   uses the auto-generated Secret "{{ include "kure.fullname" . }}-bootstrap".
   The Secret is created on first install and preserved across "helm upgrade".
   To rotate the service token, edit that Secret and restart the backend,
   agent, and security-scanner pods.

3. Configure your LLM provider (OpenAI, Anthropic, Groq, Gemini, or Ollama) in the
   Admin panel to enable AI-powered troubleshooting.
